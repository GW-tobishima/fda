# PR-75 Review Packet

## CHANGE_INTENT

PR-V1-014として、`fda merge` にGitHub merge execution adapterを追加する。既定の `fda merge` は従来どおりartifact生成のみとし、`--execute` を明示した場合だけ、Merge Gate通過済みPRをGitHubでmergeして結果receiptを残せるようにする。

## SCOPE_IN

- `fda merge --execute` と `--merge-method merge|squash|rebase` のCLI追加。
- Merge Gateが `merge_ready` かつ `auto_merge_candidate` の場合だけGitHub merge adapterを実行するprecondition。
- GitHub merge成功時の `github_merge_receipt.json` 生成。
- GitHub merge失敗時の `failure_reason` と `resume_command` を含むfail-closed receipt生成。
- GitHub merge実行時に `target_pr.head_sha` を `gh pr merge --match-head-commit` に渡し、古いreceiptで未レビューheadをmergeしない防止策。
- `--target-repo` のGitHub originと `actual_pr_url` のowner/repo照合。
- `gh pr merge` 成功後のreceipt収集失敗を、未実行ではなく `merge_executed=true` / `receipt_collection_failed` として記録する分岐。
- `merge_receipt.json`、runner explanation、artifact inventory、stdout summaryへのmerge実行結果反映。
- mock commandによる成功、失敗、precondition failure、head SHA欠落、target repo不一致、receipt収集失敗の単体テスト。

## SCOPE_OUT

- `fda merge` の既定動作を自動mergeへ変更すること。
- GitHub branch protection、merge queue、repository policyの例外承認。
- Forge PromotionDecision adapterの実装。
- ATO state adapterの実装。
- security / privacy / legal / release risk の自己承認。

## AC_EVIDENCE

- AC1: `external_pr_receipt.json` の `actual_pr_url`、`planned_pr_id`、`checks.tests`、`scope_disposition.kind=within_scope` は既存Merge Gateで必須。
- AC2: `--execute` 未指定時は `merge_execution_status=not_requested` で、GitHub mergeを実行しない。
- AC3: `--execute` 指定時は `merge_ready` / `auto_merge_candidate` / `actual_pr_url` を再確認してからadapterを実行する。
- AC4: `target_pr.head_sha` 欠落時、またはtarget repo originとPR URL owner/repoが一致しない場合はmerge実行前にfail closedする。
- AC5: 成功時は `github_merge_receipt.json` に `merge_sha`、`merge_method`、`expected_head_sha`、`merged_at`、`actor`、PR URLが残る。
- AC6: merge command成功後のreceipt収集失敗は `merge_executed=true` / `receipt_collection_failed` として記録し、再merge commandを提示しない。
- AC7: mock commandで成功、失敗、precondition failure、head SHA欠落、target repo不一致、receipt収集失敗を検証済み。
