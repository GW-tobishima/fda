# PR-78 Review Packet

## CHANGE_INTENT

#68 FU-003 として、MCP / Codex process invocation を application port と infra adapter の境界へ切り出す。
`application` は process IO を直接持たず、`CodexProcessPort` 経由で tools/list、Codex live call、git HEAD lookup を要求する。

## SCOPE_IN

- `src/application/ports.rs` に `CodexProcessPort` と `ProcessOutput` を追加する。
- Codex / MCP の result DTO を `src/domain/entities.rs` へ移し、rendering が application DTO に依存しない状態を維持する。
- `src/infra/process.rs` を追加し、`codex mcp-server` tools/list / tools/call、MCP frame IO、approval prompt fail-fast、generic process execution、git HEAD lookup を移動する。
- `src/application/implement.rs` は `CodexProcessPort` を受け取り、MCP / Codex / git HEAD 実行を port 経由にする。
- `src/lib.rs` は implement entry で `CodexMcpProcessAdapter` を注入する thin wrapper とし、MCP frame / Codex process helper を持たないようにする。

## SCOPE_OUT

- `review` / `continue` / `merge` use case 本体の module 分離は FU-004 で扱う。
- `open` / `notify` の module 分離は FU-005 で扱う。
- `lib.rs` facade gate 強化は FU-006 で扱う。
- docs の移行段階注記を完了状態へ更新する作業は FU-007 で扱う。
- merge approval、risk approval、scope approval は付与しない。
- #68 は umbrella EPIC のため、この PR 単体では close しない。
