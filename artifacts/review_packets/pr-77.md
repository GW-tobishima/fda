# PR-77 Review Packet

## CHANGE_INTENT

PR-V1-015として、`fda implement --live` にfixtureなしCodex MCP live実行の証跡を残す `live_execution_evidence.json` を追加する。
Operational V1では、unit test fixtureで成功しただけではlive実行証跡として扱わない。実際の `codex mcp-server` tools/list / codex tool call のsource、fixture使用有無、成功/失敗、PR URL、test、scope driftを分離して記録する。

## SCOPE_IN

- `fda implement --live` の出力に `live_execution_evidence.json` を追加する。
- `tools_list_source` と `codex_tool_source` を記録し、fixture経由か実Codex MCP経由かを判別できるようにする。
- fixtureを使ったlive test成功をOperational V1 proofとして扱わない `fixture_free_gate` を追加する。
- fixtureなしlive実行が失敗した場合も成功扱いにせず、fail-closed receiptと次actionを残す。
- `--live-timeout-seconds <n>` を追加し、Codex MCP live実行を有界に試行できるようにする。既定は従来相当の1800秒。
- PR-V1-015は、既存PR-V1-006 live implementer契約をOperational V1向けに検証する証跡PRであることを文書化する。

## SCOPE_OUT

- Codex MCP live implementer自体の実装対象を変更すること。
- Functional QA / Security QA / repair / merge / Forge adapter / ATO adapterの実装。
- GitHub PR作成の成功をこのPRだけで保証すること。
- Codex / GitHub / SMTP credentialの値をartifactやATOへ保存すること。

## AC_EVIDENCE

- AC1: `live_execution_evidence.json` が `mcp_tool_call_receipt.json`、`implementation_receipt.json`、`external_pr_receipt.json`、`coding_agent_thread_state.json` と並んで生成される。
- AC2: `fixture_used=true` の場合は `status=fixture_mode` かつ `fixture_free_gate.status=blocked` になり、Operational V1 proofとして通らない。
- AC3: Human Decision未解決またはdry-run未通過の場合は `tools_list_source=not_invoked` となり、Codex live toolを呼ばない。
- AC4: fixtureなし実行では `tools_list_source=codex_mcp_server`、`codex_tool_source=codex_mcp_tool_call` を記録する。
- AC5: adapter unavailable / live call failure は成功扱いにせず、`implementation_status=failed|blocked|adapter_unavailable` と次actionに残る。
- AC6: `--live-timeout-seconds` は1から86400秒までのみ受け付け、0や過大値はparse errorになる。
- AC7: helpに `fda implement --live` とtimeout optionが表示される。
- AC8: `live_execution_evidence.json` は専用schemaで検証され、fixture gateやMCP source fieldsの欠落を検出できる。
