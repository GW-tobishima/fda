## 変更概要

- FDA CLIの各 command に共通の `--ato-sync` optionを追加した。
- ATO書き戻しは明示 opt-in とし、既定では外部stateを変更しない。
- `--ato-task`、`--ato-run-id`、`--ato-backend`、`--ato-db`、`--ato-cli` でATO CLI実行先を固定できる。
- `fda start --ato-sync` で ATO task / run / checkpoint を作り、Human DecisionをATO typed decisionとして開く。
- `fda decide --ato-sync` で前回 `ato_state_receipt.json` の対応を使い、ATO decision answer / apply へ反映する。
- その他stageは主要artifactをATO evidence edgeとしてcheckpointへ戻す。
- ATO CLI/backend unavailable は成功扱いにせず、`ato_state_receipt.json` に `adapter_unavailable` と再開commandを残す。
- raw stdout / stderr全文やsecret値はreceiptに保存しない。
