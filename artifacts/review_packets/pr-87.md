# Review Packet: PR #87 / FDA V1 Codex CLI primary rebaseline

## 対象

- PR: https://github.com/msamunetogetoge/forge-delivery-agent/pull/87
- Branch: `codex/fda-v1-codex-cli-primary-rebaseline`
- Base: `main`
- Scope: FDA V1をCodex CLI primaryへ正本化し、必須 `.fda/` profile、current Codex CLI implementer handoff、Review Agent Gate、PR番号付きreview packet反映、Human merge approval境界を揃える。

## 変更概要

- `.fda/` 7ファイルprofileを作業開始前に作成し、既存profileもschema validationでfail-closedにした。
- 生成される `repo.yaml` のrepo id/name/default branch/stack/commandsをYAML文字列としてquoteし、数値・真偽値扱いされないようにした。
- `fda status` をread-onlyに戻し、存在しないartifact dir指定で `.fda/` を作らないようにした。
- `fda review` は実行済み `pr_reviewer_receipt.json` がない場合にblockedとし、PR reviewerを自動passしない。
- changed filesからATO / Forge / FDA evidence境界、handoff、review packet、Human Decision境界、UI / frontend / browser surfaceを検出し、必要時は `forge_reviewer` / `design_qa` をrequired blockedにする。
- `fda merge` は `review_agent_gate.json` と `review_agent_gate_packet.md` を必須化し、PR番号付き `artifacts/review_packets/pr-<PR番号>.md` を `scripts/check_review_agent_gate.py` で検証してからでないと進めない。
- `fda merge` は現行 `review_agent_gate_packet.md` とPR番号付きpacketの内容差分を検出し、stale reflectionをmerge前にblockする。
- `fda merge` は `external_pr_receipt.changed_files` から `forge_reviewer` / `design_qa` のrequired状態を再計算し、gate JSONの自己申告だけでは通さない。
- `fda merge` は required reviewer evidence がartifact dir内の実在ファイルであることを確認する。
- `fda merge` はFDA repoではなくtarget repoの `.fda/delivery_policy.yaml` から `auto_merge_allowed` を読む。
- `fda merge` は `external_pr_receipt.target_pr.state` が `open` でない場合、PR URLとhead SHAが揃っていてもmerge前にblockする。
- `fda merge` のHuman merge approvalは `approve` / `approve_merge` などの明示approval tokenだけを承認扱いにし、`manual_review_required` のような方針option選択をmerge approvalとして扱わない。
- `fda merge` のHuman merge approvalは `merge_approval` のようなHuman Turn理由tokenや状態tokenも承認扱いにしない。
- `fda merge` は反映済みPR packet全体ではなく `REVIEW_AGENT_GATE` セクション単位で現行packetと比較する。
- `fda merge` は `review_agent_gate.json` の `actual_pr_url` / `reviewed_planned_pr_id` を現在のPR receiptと照合する。
- `fda merge` は reviewer row の `workspace_policy=read_only` を明示検証し、reviewer evidence symlink がartifact dir外へ解決される場合はblockする。
- V1の `auto_merge_allowed: false` をmerge policyに反映し、Human merge approval receiptがある場合だけ `human_approval_granted` として `--execute` へ進める。
- `fda review` は `artifacts/runs/**`、receipt、validation report、ATO state、Forge projectionの変更をFDA証跡変更として扱い、`forge_reviewer` をrequiredにする。
- `fda review` は `forge_reviewer` が無い環境でも `qax2` / `orchestrator` fallback receiptをReview Agent Gate aggregationで受け入れる。
- `fda review` は reviewer receipt自体の `workspace_policy=read_only`、`source_mutation_allowed=false`、`actual_pr_url`、`reviewed_planned_pr_id` を照合し、古い/別PRのreceiptやwrite権限receiptをfail-closedにする。
- `fda review` は root-level `browser/` / `playwright/` 変更もdesign QA対象にする。
- `.fda/` profile作成時のschema rootはFDA本体側から解決し、外部target repoにschemaが無くてもprofile validationをfail-closeで実行する。
- `fda implement --dry-run` はtarget repoへ `.fda` profileを書く前に、出力先がtarget repo配下でないことを確認する。
- `agent_roles.yaml` schemaは `pr_reviewer` を必須化し、Review Agent Gate schemaは `qax2` / `orchestrator` fallback reviewer roleを許可する。

## Codex Review対応

- `src/application/merge.rs`: target repo policy参照、現行packetと反映済みpacketの一致確認、changed files由来のconditional reviewer再計算、reviewer evidence実在確認を追加した。
- `src/application/merge.rs`: `target_pr.state=open` と明示的なmerge approval回答だけをmerge readinessへ進める追加gateを入れた。
- `src/application/merge.rs`: `merge_approval` / `merge_approval_granted` を明示approval tokenから外し、Human Turn理由tokenではmerge approvalにならないようにした。
- `src/application/merge.rs`: `REVIEW_AGENT_GATE` セクション単位比較、current PR照合、read-only reviewer policy、canonical evidence containmentを追加した。
- `src/application/review.rs`: `artifacts/runs/**` などのFDA証跡変更で `forge_reviewer` を必須化した。
- `src/application/review.rs`: `qax2` / `orchestrator` fallback receiptとroot-level browser/playwright design QA triggerを追加した。
- `src/application/review.rs`: reviewer receiptの `workspace_policy`、`source_mutation_allowed`、`actual_pr_url`、`reviewed_planned_pr_id` を現在のPR入力と照合するようにした。
- `src/application/profile.rs`: `.fda/` 既存profile validation、YAML quote、FDA本体schema root解決を追加した。
- `src/application/implement.rs`: dry-run出力先安全確認後にtarget `.fda` profileを書くようにした。
- `src/application/status.rs`: status commandのprofile作成副作用を削除した。
- `src/rendering/merge.rs`: no-auto-merge policyとHuman merge approval済み状態をmachine-readableにした。
- `docs/standards/fda-v1/schemas/repository-profile/agent_roles_yaml.schema.json`: `pr_reviewer` を必須roleに追加した。
- `docs/standards/delivery-artifacts-v0/schemas/review_agent_gate.schema.json`: `qax2` / `orchestrator` fallback roleを許可した。
