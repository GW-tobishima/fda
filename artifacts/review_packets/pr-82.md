# Review Packet: PR #82 / FDA-ARCH-FU-005

## CHANGE_INTENT

#68 FU-005 として、`fda open` / `fda notify test` の use case を `src/lib.rs` から責務別 module へ移す。
`src/lib.rs` は既存 CLI / tests 互換の facade を残し、Output Hub の artifact 読み取りは `application/output_hub.rs`、通知要求とreceipt orchestrationは `application/notify.rs`、HTML / Markdown / SMTP本文生成は `rendering/*`、SMTPの具体I/Oは `infra/smtp.rs` が持つ。

## SCOPE_IN

- `src/application/output_hub.rs` に `fda open` の use case、artifact row / decision row / status row 生成を移す。
- `src/rendering/output_hub.rs` に Output Hub / Decision Inbox / Execution Status HTML と receipt JSON 組み立てを移す。
- `src/application/notify.rs` に `fda notify test` の use case、recipient解決、dry-run/live receipt orchestrationを移す。
- `src/rendering/notify.rs` に notification request / dry-run receipt / Human Turn notice / SMTP本文生成を移す。
- `src/infra/smtp.rs` に SMTP env読込、envelope検証、message-id生成、DNS解決、openssl / TCP送信を移す。
- `src/lib.rs` は `open_output_hub` / `notify_test` の薄いfacadeとresult re-exportを残す。
- 新規application moduleのinfra利用を `scripts/check_architecture_boundaries.py` と `docs/standards/fda-v1/architecture.md` に明示する。

## SCOPE_OUT

- UI redesign、Output Hubの見た目刷新、ブラウザ起動はしない。
- 新しい通知providerは追加しない。既存SMTP live pathの移動に限定する。
- `src/application/ato_state.rs` の既存architecture boundary違反はこのPRでは直さない。
- `src/lib.rs` facade gate強化はFU-006以降で扱う。
- merge approval、risk approval、scope approval は付与しない。
- #68 は umbrella EPIC のため、このPR単体ではcloseしない。
